## Project

Qamqor Systems website.

This is a local business website for a company in Kokshetau that provides security and engineering systems.

The website must be built locally first. Do not configure GitHub Pages, domain, hosting, deployment, CI/CD, or production infrastructure unless explicitly requested later.

- equipment sales through an office/store

The company works with:

- private clients
- houses and apartments
- shops
- offices
- warehouses
- industrial objects
- companies in Kokshetau and the region

## Design Direction

The design should be light, local, modern and trustworthy.

Avoid:

- AI-template look
- purple gradients
- neon/cyberpunk style
- excessive glassmorphism
- random 3D icons
- vague SaaS-style wording
- generic corporate filler text

Prefer:

- clean layout
- light background
- readable typography
- practical service cards
- local trust signals
- real-world business feel
- clear WhatsApp conversion path

## Content Rules

Use Russian language for the website.

Tone should be clear, human and specific.

Avoid phrases like:

- инновационные решения для вашего успеха
- индивидуальный подход
- команда профессионалов
- качество на высшем уровне

Prefer concrete wording:

- Подберём оборудование под объект
- Установим камеры и настроим доступ с телефона
- Работаем по Кокшетау и области
- Можно купить оборудование в офисе или заказать установку под ключ

## Placeholder Rules

Photos, prices and equipment lists are not final yet.

Use placeholders/examples for:

- case photos
- prices from
- equipment brands
- equipment categories

Make it easy to replace placeholders later.

Do not present example prices as final confirmed prices.

## Calculator

The site should include a preliminary cost calculator.

The calculator should collect:

- object type
- needed service
- number of cameras or points
- remote access needed or not
- location: Kokshetau or region

The result should guide the user to WhatsApp.

The calculator does not need to give an exact final price. It should create a WhatsApp message with the selected details.

## Technical Direction

Build locally first.

Do not deploy.

Do not add unnecessary backend.

Do not add authentication.

Do not add payment.

Do not build a full e-commerce cart.

Equipment page should work as a catalog/showcase with WhatsApp CTA, not as a full online store.

## Quality Checklist

Before finishing any task, check:

- all pages open locally
- navigation works
- WhatsApp buttons work
- mobile layout is not broken
- there is no horizontal scroll
- text is not generic AI copy
- placeholders are clear and replaceable
- contact data is consistent
- design feels local and trustworthy
